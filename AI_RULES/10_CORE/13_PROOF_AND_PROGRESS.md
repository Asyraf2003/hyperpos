# P1 - Proof and Progress

## Tujuan
Memastikan progres hanya didasarkan pada bukti nyata.

## Aturan
- Progres tidak boleh naik tanpa proof.
- Proof yang valid dapat berupa:
  - output command
  - file yang dibuat/diubah
  - test pass/fail
  - ADR
  - handoff
  - snapshot repo
- Setiap selesai satu step workflow, tampilkan progres persen.

## Format proof minimum
- command
- hasil
- arti hasil terhadap step aktif

## Larangan
- Jangan mengklaim "selesai" jika baru proposal.
- Jangan mengklaim test hijau jika tidak ada output.
